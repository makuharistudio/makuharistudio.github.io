This post reviews the development of my first vibe-coded Android app using AI. The final product is open-sourced as scaffolding available for professionals or experts to better expand on.

![Koe preview](/src/assets/projects/img-2026-05-koe.gif)

## The idea

I wanted a way to record speech therapy exercises for easy progress tracking, and store the data locally for privacy (i.e. on the device with no server connections). I also wanted to store information around vocal disorders and habits, particularly a convenient shopping list of foods that are good and those to avoid for vocal disorders.

When I first began the project, there was only one other app in the Google Play Store that seemed geared toward the speech therapy exercise reporting component, but it was pay-to-install and I couldn't tell if it had an informational section, so I avoided it and tried to build my own.

During my initial research for solutions, I came across a few other GitHub projects:

* AudiHealth, a browser-based Voice Pathology Detection ML model by Sahil Brid: **[GitHub](https://github.com/sahilbrid/voice-pathology-detection)**.

* Multimodal Pathological Voice Classification by Jia-Wei Liao: **[GitHub](https://github.com/jwliao1209/Multimodal-Pathological-Voice-Classification)**.

From these I expanded the idea to parse the recordings into an AI voice model trained on the **[Saarbrücken Voice Database (SVD)](https://stimmdb.coli.uni-saarland.de)** to generate metrics for conveniently displaying objective progress without needing to re-listen to past audio.

The AI models seemed passable when testing on the computer.

But ultimately, their figures were inconsistent and unreliable when testing on a mobile device. So I decided to stop pursuing accuracy and open source the project so someone better (e.g. a clinician or statistical expert in the field) could fork the project and improve it themselves, as I already got the project to a far enough point I am satisfied with.

## 1st iteration

When I first began coding the project, I chose **[React Native](https://reactnative.dev/docs/getting-started)** since it is a similar framework to React, which I use for this GitHub site. I also chose **[ONNX](https://onnxruntime.ai/docs/tutorials/mobile)** for the on-device AI model generation.

I vibe-coded this first iteration in the cheapest and most inefficient way possible: I copy-pasted file contents individually between my code editor and the web browser with **[Grok in 𝕏](https://x.com/i/grok)**.

I had over 60 conversations for this first iteration.

![Grok chat history for 1st iteration 1](/src/assets/blog/2026-05-18--01.png)

Each conversation was extremely long due to the back-and-forth copying of files. Observe how tiny the scrollbar is from one of my conversations.

![Grok chat history for 1st iteration 2](/src/assets/blog/2026-05-18--02.png)

The inefficiency came from cycles of Grok breaking one thing after fixing another. Occasionally I examined the code myself and made suggestions to Grok to break free of those loops. I attribute this problem to the limited context window using Grok via X.

As the project got close to completion (it only needed styling adjustments), a **[critical exploit was identified for the Metro server](https://github.com/advisories/GHSA-399j-vxmf-hjvr)**, which is the app emulation platform of the React Native ecosystem. This also led to a fundamental architectural change in React Native that required package dependencies to update.

I tried to rebuild the project, but was unable to use any package to record audio. I tried around 5, to the best of my memory. After waiting for an alternative that didn't come, I realised my fundamental assumption was incorrect.

There was no need to rely on the React Native as a tertiary framework at all. With AI, you can choose the framework most efficient for the platform, regardless of your own programming language preferences.

So I scrapped the first React Native + ONNX iteration, and started a second using Android Kotlin + LiteRT.

Here are the repos for the first iteration. These will be set to private since they are not useful now. Possibly deleted in the future too.

1. **[Python scripts to generate ONNX AI models](https://github.com/makuharistudio/dysphonia-analysis)**.

2. **["React Native Voice Analysis" React Native app](https://github.com/makuharistudio/react_native_voice_analysis)**.


## 2nd iteration

The first iteration was designed for my own use and so only supported 1 gender. To increase the chance of someone forking the project, I made the second iteration support male and female, since both are supplied in the **[Saarbrücken Voice Database (SVD)](https://stimmdb.coli.uni-saarland.de)**.

To start, I used **[Android Studio with Gemini](https://developer.android.com/studio/gemini/get-started)**. But after building the first few screens for basic navigation, I lost access to Gemini supposedly due to authentication issues, but I suspect I hit a free-to-use limit.

![Google Gemini access lost](/src/assets/blog/2026-05-18--03.png)

So I bought credits from xAI to try **[Grok with a code editor](https://docs.x.ai/developers/advanced-api-usage/use-with-code-editors)** and see if that would up my efficiency. I also installed the **[Cline AI coding agent extension](https://marketplace.visualstudio.com/items?itemName=saoudrizwan.claude-dev)** as it was the first listed on the **[xAI page at the time](https://web.archive.org/web/20260501131756/https://docs.x.ai/developers/advanced-api-usage/use-with-code-editors#using-grok-code-models-with-cursor)**.

![Wayback Machine for xAI Cline recommendation](/src/assets/blog/2026-05-18--04.png)

However, after finishing and releasing the project, I noticed xAI updated their page such that Cline was no longer listed. Before that point and throughout the development process, I noticed Cline was about 0.2 or 0.3 versions behind the latest xAI models. 

![VSCodium with Cline](/src/assets/blog/2026-05-18--05.png)

So I found myself falling back to copy-pasting code with **[Grok in 𝕏](https://x.com/i/grok)** as it uses the latest models which were better for more advanced code issues I couldn't solve via Cline.

I had over 40 Grok in 𝕏 conversations with this second iteration, on top of many via Cline.

![Grok chat history for 2nd iteration](/src/assets/blog/2026-05-18--06.png)

In that sense, I wasn't surprised xAI pulled Cline off their page. 

![xAI Use with Code Editors](/src/assets/blog/2026-05-18--07.png)

If Cline wasn't keeping up to xAI's models, it would only be a liability for xAI, as users would be pointing to older models which are cheaper, and using up xAI's processing with the back-and-forth entailed by older models.

That said, I enjoyed agentic AI for removing so much friction and am grateful to the Cline team. Though I formatted my machine after completing the project. Whilst Cline is not as intense as OpenClaw, AI agentic tools can become severe security risks.

### UI design

My UI design skills are limited.

Though I understood I had to hold back on elaborate designs for mobile apps, and keep it simple. Still, I added male and female avatars for the main menu and exercise-related screens to make it a little engaging.

![Male and female app avatars 1](/src/assets/blog/2026-05-18--08.png)

### Making video tutorial YouTube shorts

Creating video tutorials was an easy process. My Android device has a built-in screen recorder, and I used **[Da Vinci Resolve](https://www.blackmagicdesign.com/products/davinciresolve)** to edit the **[YouTube shorts](https://www.youtube.com/watch?v=3flgXDR4dvs&list=PLy6h2tqAri6AaWhOtFZp7rrDY1yLxUPAM)** as quick instructional guides.

I was relieved to find that xAI provides a **[Grok Text-To-Speech (TTS) playground](https://console.x.ai/playground/voice/text-to-speech)**. I selected the British voice named "Olivia" because the resemblance was striking how it sounded nearly identical to one of my previous work colleagues. It has a warm elegance to it which is excellent for the learning process.

![xAI Grok TTS](/src/assets/blog/2026-05-18--10.png)

## Total cost

The largest cost for this project was time, since I essentially had to do it twice. If you separate the model training and testing from the Android app as different projects, then I arguably spent the time working on 4 projects. I certainly have 4 repos to show for it.

Though the financial cost for the second iteration of this project using paid Grok with an agentic-powered code editor, was about $45 USD, as per screenshot below.

![xAI costs](/src/assets/blog/2026-05-18--11.png)

Bear in mind, I took advantage of Cline's ability to use separate models for reasoning versus coding, to lower costs. I used Grok 4.1 Fast for reasoning, the latest available at the time was 4.3 but Cline did not facilitate this. I also used Grok Code Fast 1 for the coding. Both turned out to be **[officially deprecated](https://docs.x.ai/developers/migration/may-15-retirement)** a couple of days after I finished the project.

Just in time.
