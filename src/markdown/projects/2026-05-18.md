This is an experimental open-source Proof-of-Concept (PoC) Android app for recording vocal exercises and tracking progress using on-device AI.

This project is provided as scaffolding to help qualified speech therapists, voice clinicians, and researchers quickly build and customise their own practice-specific tools.

The app lets users:
* Browse 2 sections of articles relating to vocal disorders and vocal habits.
* Browse and select speech therapy exercises.
* Record audio, photo, and/or video for each exercise.
* Track progress with a calendar heatmap.
* Run experimental on-device AI analysis on audio files to generate vocal health metrics whose values may be inaccurate, biased, or unreliable.

It is designed to be easily expandable using very simple markdown files to add new articles and exercises, and add new AI models for other vocal disorders.

There are two GitHub repositories for this project, both vibe-coded with Grok:

1. **[https://github.com/makuharistudio/dysphonia-analysis-general](https://github.com/makuharistudio/dysphonia-analysis-general)** - Python scripts that extract and convert healthy and impaired vocal files from the **[Saarbrücken Voice Database (SVD)](https://stimmdb.coli.uni-saarland.de)**, then trains them into **[LiteRT](https://github.com/google-ai-edge/litert)** AI models that run on mobile devices for offline inference.

2. **[https://github.com/makuharistudio/koe](https://github.com/makuharistudio/koe)** - Proof-of-Concept (PoC) Android Kotlin app scaffolding used to interface with said AI model.

Both are open-sourced under **[MIT License](https://opensource.org/license/mit)**. So please feel free to **[fork](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo)** the project.

**Technologies**:
* **[Kotlin](https://developer.android.com/kotlin)** + Jetpack Compose for App framework.
* **[Room](https://developer.android.com/training/data-storage/room)** + **[SQLite](https://developer.android.com/training/data-storage/sqlite)** database.
* **[Google Tink](https://developers.google.com/tink)** encrypted media files and database fields.
* Python libraries: **[NumPy](https://numpy.org)**, **[scikit-learn](https://scikit-learn.org/stable/index.html)**, **[Librosa](https://librosa.org/doc/latest/index.html)**, and **[Parselmouth](https://parselmouth.readthedocs.io/en/stable)**
* **[LiteRT](https://github.com/google-ai-edge/litert)** (formerly TensorFlow Lite) for on-device inference.
* **[Markdown](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax)**-driven content system (easy to customise).

For further information on how this app was made, please see my blog post: **[https://makuharistudio.github.io/#/blog/vibe-coding-an-android-app-with-on-device-inference-ai-model](https://makuharistudio.github.io/#/blog/vibe-coding-an-android-app-with-on-device-inference-ai-model)**.
